#### Round 2: Technical Interview - JavaScript Deep Dive

**Core JavaScript Questions:**
```javascript
// 1. Explain Event Loop
// Answer: Single-threaded, call stack, callback queue, microtask queue

// 2. What's the output?
console.log(1);
setTimeout(() => console.log(2), 0);
Promise.resolve().then(() => console.log(3));
console.log(4);
// Output: 1, 4, 3, 2 (microtasks before macrotasks)

// 3. Closures
function outer() {
    let count = 0;
    return function inner() {
        count++;
        return count;
    }
}
const counter = outer();
counter(); // 1
counter(); // 2

// 4. this keyword
const obj = {
    name: 'Avinash',
    greet: function() { console.log(this.name); },
    greetArrow: () => { console.log(this.name); }
};
obj.greet(); // 'Avinash'
obj.greetArrow(); // undefined (arrow doesn't bind this)

// 5. Promises vs Async/Await
async function fetchData() {
    try {
        const response = await fetch(url);
        const data = await response.json();
        return data;
    } catch (error) {
        console.error(error);
    }
}

// 6. Debounce Implementation
function debounce(fn, delay) {
    let timeoutId;
    return function(...args) {
        clearTimeout(timeoutId);
        timeoutId = setTimeout(() => fn.apply(this, args), delay);
    }
}

// 7. Throttle Implementation
function throttle(fn, limit) {
    let inThrottle;
    return function(...args) {
        if (!inThrottle) {
            fn.apply(this, args);
            inThrottle = true;
            setTimeout(() => inThrottle = false, limit);
        }
    }
}
```

**Topics to Master:**
- [ ] Hoisting (var vs let vs const)
- [ ] Prototypes and inheritance
- [ ] Event delegation and bubbling
- [ ] Memory leaks and garbage collection
- [ ] ES6+ features (spread, destructuring, modules)

```jsx
        const stored = localStorage.getItem(key);
        return stored ? JSON.parse(stored) : initialValue;
    });
    
    useEffect(() => {
        localStorage.setItem(key, JSON.stringify(value));
    }, [key, value]);
    
    return [value, setValue];
}

// 6. Why keys are important
// Answer: Help React identify which items changed, added, or removed
// Never use index as key when list can reorder
```

### ‚ùå Interview Mistakes to Avoid

| Mistake | Why It's Bad | What to Do Instead |
|---------|--------------|-------------------|
| Saying "I don't know" immediately | Shows lack of problem-solving | "I haven't used this directly, but based on my understanding of [related concept]..." |
| Coding without discussing approach | Interviewers can't evaluate your thinking | Explain your approach first, get confirmation, then code |
| Ignoring edge cases | Shows incomplete thinking | Always ask: "What about empty input? Null? Very large values?" |
| Not asking clarifying questions | Assumptions lead to wrong solutions | Ask about constraints, expected input/output, scale |
| Speaking negatively about past work | Red flag for team culture | Focus on what you learned, not what was wrong |
| Overselling yourself | Easily caught in follow-up questions | Be honest about your experience level |
| Not admitting mistakes | Shows inability to learn | "You're right, I missed that. Let me reconsider..." |
